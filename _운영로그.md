## 자동 실행 모드 (Auto-Pilot) 운영 규칙

1. **각 에이전트 = 한 페르소나로 진입** — 시스템 프롬프트의 정신을 따른다.
2. **모듈화** — 산출물은 파일로 저장, 다음 단계는 핵심 요약만 참조.
3. **비가역 액션 금지** — GitHub push, 외부 신청서 제출 등은 사용자 승인 후.
4. **한계 만나면 즉시 기록** → 다음 단계가 그 기록 참조.
5. **답변 본문 = 보고만** — 풀 산출물은 파일로.
6. **확인 필요 사항** 은 산출물에 명시 → Reviewer/Analyst 단계에서 해결 시도.

## 발생 문제 / 개선 (이 자리에 누적)

### 문제 1 — 한국 공공데이터에 AI 변수가 너무 늦게 등장
- **발견 단계**: Reviewer (04_문헌리뷰.md)
- **영향**: H1·H2 시계열 검증 거의 불가, 2023~ 횡단으로 다운그레이드 강제
- **개선**: Planner v2에서 SH7을 SH7a(가능)/SH7b(다운그레이드)로 분리. 4종 모두 한계 명시.

### 문제 2 — 창의성 직접 측정 변수 부재
- **발견 단계**: Critic (03_비판검토.md, 결함 ①)
- **영향**: H2 검증의 구성타당도 약화
- **개선**: Planner v2에서 H2 영역을 "창의적 표현 활동의 빈도·다양성"으로 다운그레이드 + 4종 본문에 한계 명시 + 후속 1차 자료 연구 권고.

### 문제 3 — 9~12세 데이터 빈약
- **발견 단계**: Critic (결함 ⑦)
- **영향**: 청소년기본법 양 끝단 중 하나가 분석 부족
- **개선**: Planner v2에서 "13~24세 핵심, 9~12세 보조" 명시. 9~12세는 여가부 청소년종합실태로 보강.

### 문제 4 — 마이크로데이터 신청·승인 일정 = 논문 일정 결정
- **발견 단계**: Reviewer
- **영향**: 정밀 분석 1~3개월 지연
- **개선**: 1차 발행을 "윤곽"으로 명시 + Phase 2(정밀)로 v2 미루기.

### 문제 5 — Claude 학습 cutoff 2026-01 → 인용 정확도 한계
- **발견 단계**: Reviewer
- **영향**: 인용 수치·연도 1차에서 정확히 못 박음
- **개선**: 4종 본문에 "GPT CLI 웹검색 재실행 필수" 명시 + 사용자 검증 항목으로.
