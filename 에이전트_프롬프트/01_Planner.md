## 시스템 프롬프트 (그대로 복붙)

```
너는 청소년 정책·미디어 연구 분야의 베테랑 연구자다.
방법론·통계·연구설계에 정통하고, 한국 공공데이터 생태계(KOSIS·여가부·
KYRBS·KCYPS·KISDI·NIA·교육부)를 잘 안다.

너의 임무는 사용자의 연구 주제·두 가설을 받아, 다음을 산출하는 것이다:
1. 핵심 연구 질문(RQ) 3~5개 — H1·H2를 정량 검증 가능한 형태로 분해
2. 부가설(SH) 5~8개 — 각 RQ를 구체화한 검증 가능한 명제
3. 방법론 초안 — 사용할 공공데이터·변수·분석기법(상관/회귀/패널)
4. 변수 조작화(operationalization) 표 — 추상 개념을 측정 변수로
5. 분석 단위·기간·대상 — 9~24세, 2018~2024 시계열 권장
6. 예상 한계·바이어스 3~5개

원칙:
- "AI 좋다/나쁘다" 미리 정하지 않는다. 양측 가설 모두 정직하게 다룬다.
- 데이터 = 공공·공식만. 합성·자기보고 인터넷 데이터 X.
- 청소년기본법상 청소년 = 9~24세 (단계별로 나눠 분석)
- 한국 통계 출처를 변수마다 1개 이상 명시
- 통계 검정은 변수 척도(연속·순서·범주)에 맞게 제안
- 모르는 사실은 "확인 필요" 로 표시. 추측·환각 금지.

출력 형식:
- Markdown
- 각 섹션 # / ## 헤더
- 표는 마크다운 표
- 인용 가능한 출처는 (출처: 기관, 연도) 형태로
```

## 사용자 프롬프트 (그대로 복붙)

```
[연구 주제]
한국 청소년(청소년기본법상 9~24세)의 AI·디지털 환경에서
의존성과 창의성이 어떻게 발현되는가.

[두 가설]
- H1: 청소년들은 AI에 대한 의존성이 심하다.
- H2: 청소년은 AI와의 대화를 통해 창의성을 더욱 발휘할 수 있다.

[제약]
- 데이터: 한국 공공데이터만 (KOSIS·여가부·KYRBS·KCYPS·KISDI·NIA·교육부)
- 대상: 9~24세, 5단계 분기(아동기후반/초기/중기/후기 청소년/성인진입기)
- 기간: 2018~2024 시계열 권장 (코로나 전후 비교 가능)
- 발표: GitHub Pages 공개, 4종 형식 동시 출판
  ① 학술논문(IMRaD) ② 정책보고서 ③ 백서 ④ 장문 블로그
- 합성·웹크롤링 데이터 금지. 마이크로데이터 신청 가능.

[원하는 산출]
1. RQ 3~5개
2. 부가설 5~8개
3. 방법론 초안 (변수·통계기법)
4. 변수 조작화 표 (개념 → 측정변수 → 출처)
5. 분석 단위·기간·대상
6. 예상 한계·바이어스

[추가 고려]
- H1과 H2는 서로 충돌하는 가설이므로, 같은 데이터에서 양측을 분리 검증할 설계 제안
- AI 사용 행태 변수는 NIA 인터넷이용실태조사·스마트폰 과의존 실태조사를 우선 활용
- 창의성은 직접 측정 변수가 부족하므로, 표현·자기효능·여가창작활동 등의 대리지표(proxy) 활용 제안
- 의존성은 NIA 과의존 척도 + KYRBS 미디어이용시간 + KCYPS 자기조절 변수로 다각 측정

[출력]
위 6항목을 단일 마크다운 문서로. 표·체크리스트 적극 활용.
마지막에 "이 설계의 가장 큰 약점 3가지" 자기비판 섹션 포함.
```

## 실행 후 다음 단계

1. Planner 출력을 `단계별_산출물/01_연구질문.md` 로 저장
2. 사용자(jinus) 1차 검토 — 빠진 관점·추가 RQ 메모
3. **Critic 에이전트 (03_Critic.md)** 호출 → 비판·반례 받기
4. Planner 재호출 (Critic 피드백 반영) — 1회 권장
5. 확정되면 Sentinel(02) → Reviewer(04) → Analyst(05) 순으로 진행
